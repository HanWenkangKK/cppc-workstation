# 任务执行规则

每次任务启动前，你必须：

## 步骤 1：深度理解
1. 输出你对业务场景（神经康复/CPPC）的理解。
2. 识别该任务是否会破坏现有的“单体架构”或“数据流向”。

## 步骤 2：契约确认（不要直接写代码）
1. 给出涉及的数据库字段变更。
2. **必须给出接口的 Request/Response 样例 JSON**。
3. 待我确认该 JSON 结构能支持前端“点击修改”的需求后，再进行编码。

## 步骤 3：最小实现
1. 仅修改实现功能所必须的代码。
2. 保持代码的可测试性。

## 输出要求
完成后，必须列出：
- 变更文件清单。
- 接口测试建议（如 cURL 命令）。
