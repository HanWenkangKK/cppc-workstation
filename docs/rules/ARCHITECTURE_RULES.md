# 架构约束（必须严格遵守）

## 技术栈
- 后端：Spring Boot 3.x
- 数据库：MySQL 8.x
- AI：通过 API 调用 Coze 工作流（非本地模型）

## 架构原则
1. **微服务架构**：代码需解耦，为后续扩展提供基础，但是在MVP阶段要重点实现核心功能。
2. **中转站模式**：后端负责权限验证、数据持久化及 Context 组装，不处理 RAG 逻辑。
3. **Controller -> Service -> Mapper/Repository** 标准三层。

## 数据原则
- **禁止透传**：AI 返回的数据必须经过后端校验并持久化，禁止前端直接通过后端代理访问 AI。
- **状态留痕**：所有的修改必须记录原值与现值。

## 代码原则
- 优先简单可运行，严禁过度设计。
- 引入新依赖必须说明理由。
