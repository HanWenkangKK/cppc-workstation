# 爱可复 & 华西 CPPC 神经康复 AI 智能工作站 - 前端架构与 UI 设计规范

## 1. 项目背景与设计原则
本项目为医疗康复垂直领域的 AI 智能工作站前端。开发需严格遵守以下原则：
* **视觉规范**：主色调采用医疗绿/静谧蓝；辅助色使用警示红（高风险预警）和活力橙（待办/非严重提示）；背景以中性灰/纯白为主。
* **交互规范**：核心业务流操作 ≤5 步；高风险操作（如禁忌证处理）强制拦截并二次确认；多端适配（PC 医生工作站 + 平板查房/谈单）。
* **角色权限**：严格区分新人治疗师（强制带教提示、无终审权）、资深治疗师（全量检索、终审权）和医疗负责人（全量管理看板）。

## 2. 技术栈与版本要求
* **核心框架**：Vue 3
* **构建工具**：Vite
* **UI 组件库**：Element Plus (需定制医疗主题)
* **状态管理**：Pinia
* **路由管理**：Vue Router 4
* **可视化图表**：ECharts / DataV (大屏专用)
* **网络请求**：Axios (需封装全局拦截器与 Token 管理)
* **其他辅助**：`pdfmake` + `Print.js` (报告导出)、WebRTC (投屏对接)

## 3. 核心目录架构
系统采用高度解耦的模块化设计，请严格按照此目录结构生成代码：
```text
src/
├── assets/                  # 静态资源与 Element Plus 主题覆盖
├── components/              # 全局公共组件 (VoiceInput, RiskAlert, PdfExport, CPPCLogicTip)
├── layout/                  # 页面布局 (侧边栏、顶栏、主内容区)
├── router/                  # 基于角色的动态路由配置
├── store/                   # Pinia 状态 (user.js, patient.js)
├── api/                     # Axios 模块化接口 (按后端微服务划分)
├── views/                   # 核心业务页面
│   ├── dashboard/           # 首屏：智能运营与临床工作台
│   ├── assessment/          # 评估录入与语音辅助
│   ├── report/              # 智能报告展示 (专业/通俗双版本)
│   ├── rehab-plan/          # 康复方案与赋能包
│   └── ...                  # 其他管理页面
├── utils/                   # 工具类 (request.js, permission.js, rtc.js)
└── main.js

```

## 4. 核心页面原型要求：智能运营与临床工作台 (首屏)
这是登录后的首屏（Landing Page），采用 Layout 左右分栏结构（侧边菜单 + 顶部 Header + 底部 Main Content）。

顶部数据看板：展示提效数据（节省时长）、业务数据（今日预约、转化率）和质控合规率。使用 ECharts 渲染趋势图。

左侧待办中心：列表展示“待接诊患者”、“待审核报告”及“AI 智能预警消息”。

右侧快捷操作：醒目的“新患者评估录入”大按钮，以及知识库、同类标杆案例库的快捷检索入口。

底部/浮窗监控：针对管理层展示新人成长曲线，展示系统/AI 接口当前的健康状态。

加载状态：页面数据加载时，强制使用 Element Plus 的 Skeleton 骨架屏组件。

## 5. 严格的 AI 代码生成约束 (Coding Constraints)
在根据本规范生成任何前端代码时，必须绝对遵守以下规则：

强制语法：Vue 3 组件必须采用 Composition API 和 <script setup> 语法糖，严禁使用 Options API。

状态管理：严禁在组件内过度使用 Provide/Inject 传递复杂状态，全局业务状态（如当前接诊的患者上下文）必须统一存放于 Pinia 的 patient.js 中。

组件拆分：如果单文件组件 (SFC) 超过 300 行代码，必须将其逻辑拆分为独立的子组件或抽取复用的 Composables 函数（存放在 src/composables/）。

Mock 数据：在后端接口未就绪时，所有列表和表单必须提供结构清晰的 Mock 数据，以保证页面可独立运行和预览。

样式隔离：所有组件的样式必须添加 scoped，如果有针对 Element Plus 样式的覆盖需求，必须集中定义在 src/assets/theme，或者在此页面用 :deep() 选择器。
