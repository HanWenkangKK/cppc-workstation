# 编码规则

## 基本规则
1. **类型安全**：必须定义明确的 DTO 类。
2. **外部接口封装**：所有与 Coze 的通信必须封装在 `RemoteAiService` 中，使用 Feign 或 RestTemplate。
3. **接口幂等性**：报告保存接口需支持局部覆盖更新。

## 代码风格
- 使用清晰的英文命名。
- 强制使用 Lombok 简化代码。
- 异常处理：必须实现全局异常处理器，返回统一格式：`{code, msg, data}`。

## 局部修改逻辑
- 必须提供 `PATCH` 接口，支持通过 `element_id` 修改报告 JSON 数组中的特定项。
